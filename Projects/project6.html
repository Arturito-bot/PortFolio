Argos : Outil d'Automatisation Visuelle Multi-Profils

Un utilitaire d'automatisation en Python conçu pour surveiller plusieurs zones de l'écran et déclencher des actions (clavier/souris) en réponse à des changements visuels.

Démo de l'Interface

Voir le code sur GitHub

Technologies Utilisées

Python OpenCV Tkinter Threading mss (Capture) PyAutoGUI

Fonctionnalités Principales

Gestion de Profils Multi-Zones

Le cœur du système. Chaque profil est une zone de détection indépendante. L'utilisateur peut en créer et configurer autant que nécessaire pour surveiller plusieurs points de l'écran simultanément.

Configuration en Temps Réel

Une fenêtre de visualisation (OpenCV) permet de positionner et redimensionner la zone de détection au pixel près. La zone de capture se centre dynamiquement sur la zone de travail pour une configuration aisée.

Réglages Fins par Profil

Chaque profil possède ses propres paramètres : action (touche clavier ou clic), seuil de sensibilité (Delta), et mode de déclenchement (continu ou "one-shot" pour une action unique).

Architecture et Défis Techniques

Défi 1 : Latence de Capture

Problème : `pyautogui.screenshot()` était un goulot d'étranglement, trop lent pour une détection réactive.

Solution : Intégration de `mss` (Monitor Screen Shot), qui utilise les API natives du système pour des captures ultra-rapides, réduisant drastiquement la latence.

Défi 2 : GUI non réactif

Problème : Une boucle de détection intensive ("worker") dans le thread principal gèlerait l'interface Tkinter.

Solution : Architecture Multi-Thread. Le Thread Principal gère la GUI (Tkinter) et les configs. Un Thread Secondaire ("worker") gère 100% des opérations coûteuses.

Défi 3 : Sécurité des Threads

Problème : Le "worker" ne peut pas modifier la GUI Tkinter directement sans risque de conflits ("race conditions").

Solution : Les mises à jour de la GUI (ex: passage en pause) sont postées de manière sécurisée sur le thread principal via `root.after()`.
